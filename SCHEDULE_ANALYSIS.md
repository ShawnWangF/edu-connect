# 蘇州學校交流排程表業務邏輯分析

## 排程表結構
- **橫軸（列）**：日期（7/1 - 7/15，共15天）
- **縱軸（行）**：各批次/學校分組（P1-P8 小學，S1-S5 中學）

## 色塊顏色含義
| 顏色代碼 | 含義 |
|---------|------|
| `#9DC3E6`（淺藍） | 抵達深圳（深圳住宿日） |
| `#BDD7EE`（較深藍） | 抵達香港（香港住宿日） |
| `#1F4E79`（深藍） | 交流日（★標記，在港澳學校進行交流） |
| `#FFE699`（黃色） | 過關日（深圳↔香港，住宿城市變動） |
| `#FFFFF2CC`（淺黃） | 空白/未排程 |
| `#FCE4D6`（淺橙） | 批次4（S2-S5, P8）的行程色塊 |

## 批次結構
- **批次1**：P1（太倉市實驗小學+相城區玉成+湘城）+ P2（常熟塔前+蘇州市實驗）→ 香港進
- **批次2**：P3（金雞湖）+ P4（星灣+文正）→ 深圳進
- **批次3**：P5（蘇苑+星洲+蘇大二實）+ S1（張家港一中）+ P6（寶帶+南師大吳江）+ P7（文萃+昆山震川）→ 混合
- **批次4**：S2（西交大附中+蘇中園區）+ S3（立達+滄浪）+ S4（東沙湖+相城中）+ S5（黃橋+城西）+ P8（高新區實驗）

## 日程模式（以批次1 P1/P2 為例）
1. **7/1**：抵達香港（淺藍=深圳？不對，應是香港）
2. **7/2**：交流日（深藍）
3. **7/3**：香港住宿（較深藍）
4. **7/4**：過關日（黃色）→ 從香港到深圳
5. **7/5**：深圳住宿（淺藍）
6. **7/6**：離開（空白/返程）

## 核心約束條件
1. **交流日（★）**：必須在港澳學校可交流日進行，不可隨意調整
2. **過關日**：統計當晚住宿城市變動，影響酒店預訂
3. **批次航班**：同批次學校乘坐同一航班，調整時需整批移動
4. **住宿統計**：每日香港/深圳住宿人數，用於酒店預訂

## 需要實現的功能
1. **排程總覽**：橫軸=日期，縱軸=批次，色塊顯示每日狀態
2. **色塊拖拽**：整體移動某批次的行程（所有色塊一起移動）
3. **交流日標記**：特殊標記（★），可單獨調整
4. **住宿統計**：自動計算每日香港/深圳住宿人數
5. **航班統計**：自動計算每日抵達/離開航班數
6. **衝突提醒**：當交流日與港澳學校不可交流日衝突時提醒

## 數據模型設計
```
trips（交流項目）
  - id, name, year, startDate, endDate

batches（批次）
  - id, tripId, batchCode（P1/P2/S1等）, type（小學/中學）
  - headcount, startCity（香港進/深圳進）, batchGroup（批次1/2/3/4）
  - flightIn, flightOut（航班信息）

schools（學校）
  - id, name, city, type

batchSchools（批次學校關聯）
  - batchId, schoolId, studentCount

scheduleBlocks（日程色塊）
  - id, batchId, date, blockType（sz_arrive/hk_arrive/exchange/border_cross/departure）
  - isExchangeDay（是否交流日★）, notes

exchangeDays（港澳學校可交流日）
  - id, hkSchoolId, date, isAvailable
```
